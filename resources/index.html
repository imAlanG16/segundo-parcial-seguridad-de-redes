VPN Site-to-Site (IKEv1 + GRE)

La conectividad entre la LAN Oeste (Clientes `14.3.10.0/24`) y la LAN Este (Servidores `14.3.20.0/24`) se realiza a través de un túnel lógico GRE encapsulado en una política IPSec basada en IKEv1 (IPs de túnel `10.0.0.1` y `10.0.0.2`). Esto garantiza cifrado AES-256 de extremo a extremo para los dispositivos finales.

Autenticación Centralizada RADIUS

Se integra Active Directory Domain Services (AD DS) con Network Policy Server (NPS, `14.3.20.2`) para actuar como servidor RADIUS. La administración de R-Oeste y R-Este se autentica remotamente mapeando las políticas de NPS de niveles 15, 10 y 1 a los grupos de AD.

Seguridad de Capa 2 (Mitigación)

La electrónica de red está asegurada contra ataques clásicos. Se implementó Port-Security (limite 2 MACs sticky, shutdown ante violaciones), DHCP Snooping para mitigar servidores atacantes, Dynamic ARP Inspection (DAI) y BPDU Guard con PortFast.

Restricción de Acceso por ACL

Para proteger el Windows Server (`14.3.20.2`), se aplica una lista de acceso de rango extendido en la entrada del túnel VPN de R-Este. La ACL permite únicamente el tráfico HTTP (puerto TCP 80) e ICMP (Ping) desde la LAN Oeste, bloqueando RDP, SSH o SMB.
